# Keep or Let Go / 留不留 项目说明

## 1. 项目定位

这是一个中文单件衣服判断工具。

用户上传试穿照或衣服照片，选择判断目的和单品类型，AI 根据图片与表单信息生成判断结果。

当前产品不是泛泛的 AI 穿搭助手，而是围绕「单件衣服」做三类判断：

* 要不要留下 / 退掉
* 这件适不适合我
* 这件应该怎么搭

核心目标是帮助用户减少衣橱纠结，看清一件衣服的真实价值、适配度、搭配方式和闲置风险。

---

## 2. 当前主流程

### 首页

路径：

`/`

功能：

* 展示产品定位
* 展示适用场景
* CTA 进入判断页
* 从首页点击「开始判断一件衣服」时，进入 `/decision-helper?reset=1`
* `reset=1` 表示开始新判断，会清空旧图片、旧表单、旧结果

---

### 判断页

路径：

`/decision-helper`

主要功能：

* 上传图片
* 选择判断目的
* 选择单品类型
* 可选填写纠结点、第一感受、类似单品、使用场景、价格感受、补充说明

必填项：

* 图片
* 判断目的
* 单品类型

判断目的目前为：

* 要不要留下 / 退掉
* 这件适不适合我
* 这件应该怎么搭

交互逻辑：

* 从首页进入时带 `reset=1`，清空旧数据
* 从结果页点击「修改表单重新判断」时，不带 reset，恢复旧表单和图片预览
* 从结果页点击「再判断一件」时，清空旧数据重新开始
* 可选信息展开后，提交按钮位于底部，不需要用户再滑回上方

---

### API

路径：

`app/api/evaluate/route.ts`

功能：

* 接收图片 base64 data URL 和表单信息
* 调用 Kimi / Moonshot API
* 要求模型返回固定 JSON 结构
* 如果第一次 JSON 解析失败，会用 compact prompt 重试一次

当前 Kimi 关键配置：

* `thinking: { type: "disabled" }`
* `temperature: 0.6`

不要随意改这两项。之前已经验证过：

* `kimi-k2.6` 需要 `temperature: 0.6`
* 如果不开启 `thinking: { type: "disabled" }`，可能出现 content 为空或 token 被 reasoning_content 占用的问题

环境变量在 `.env.local`：

* `MOONSHOT_API_KEY=真实 Key`
* `KIMI_BASE_URL=https://api.moonshot.cn/v1`
* `KIMI_MODEL=kimi-k2.6`

---

### 结果页

路径：

`/result`

功能：

* 从 localStorage 读取 `keepOrLetGoForm` 和 `keepOrLetGoResult`
* 展示 AI 判断结果
* 展示上传图片
* 展示搭配建议
* 展示最终建议
* 展示详细分析折叠区
* 支持反馈按钮
* 支持重新判断

结果页已针对三种判断目的做差异化：

#### 要不要留下 / 退掉

重点：

* 保留价值
* 闲置风险
* 重复度
* 价格感受
* 是否值得继续占衣橱位置

#### 这件适不适合我

重点：

* 颜色适配
* 版型比例
* 风格适配
* 是否显高显瘦
* 是否适合用户气质和使用场景

#### 这件应该怎么搭

重点：

* 顶部大结论展示具体搭配方向
* 中段展示搭配可行度
* 不再把主结论写成「有条件留下」或「有搭配空间」
* 搭配方案是结果核心

## 3. localStorage 使用

当前使用的 key：

* `keepOrLetGoForm`
* `keepOrLetGoResult`
* `keepOrLetGoFeedback`

逻辑：

* 表单提交前保存 `keepOrLetGoForm`
* API 成功后保存 `keepOrLetGoResult`
* 反馈按钮保存 `keepOrLetGoFeedback`
* 首页进入判断页时带 reset，会清空这些 key
* 修改表单重新判断时保留 `keepOrLetGoForm`

## 4. 当前已验证功能

已验证：

* 首页 CTA 可以进入判断页
* 从首页进入会清空旧表单
* 判断页可以上传图片

* API 可以正常返回结果
* 结果页可以正常展示
* 三个判断目的逻辑都能跑通
* 「这件应该怎么搭」不会再用留不留作为主结论
* 「修改表单重新判断」可以恢复旧图片和选项
* 「再判断一件」会清空旧内容
* 「返回首页」后再开始判断，会进入空白表单
* 已加入错误兜底：/result 空状态、localStorage 损坏清理、图片格式校验、图片大小校验、API 超时提示、网络失败提示、生成中防重复点击

## 7. 下一阶段建议

下一阶段建议优先级：

* 首页视觉层级

* 「适不适合我」是否过度偏留不留
